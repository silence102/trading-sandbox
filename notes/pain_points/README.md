# 투자 Pain Points (문제점 및 개선 사항)

## 목적

이 폴더는 **실제 투자 과정에서 경험한 어려움, 문제점, 개선 희망 사항**을 기록합니다.

문제 정의 단계에서 HTS를 활용한 실제 투자와 모의투자를 병행하며 경험한 내용을 바탕으로, 기능 정의 단계에서 해결해야 할 과제들을 도출하기 위한 자료입니다.

## 기록 원칙

1. **구체적으로 작성**: 언제, 어떤 상황에서, 무엇이 불편했는지
2. **영향도 명시**: 해당 문제가 투자에 어떤 영향을 미쳤는지
3. **희망 해결책 제시**: 이상적으로 어떻게 해결되면 좋을지

## 기록 템플릿

새로운 문제점을 기록할 때 아래 템플릿을 참고하세요:

```markdown
# [문제점 제목]

## 상황
- 언제/어떤 상황에서 발생했는지

## 문제점
- 구체적으로 무엇이 불편하거나 어려웠는지

## 영향
- 투자 의사결정에 미친 영향

## 희망 해결책
- 이상적으로 어떻게 해결되면 좋을지

## 우선순위
- [ ] 높음 (즉시 해결 필요)
- [ ] 중간 (개선되면 좋음)
- [ ] 낮음 (있으면 좋음)
```

## 문제점 카테고리

| 카테고리 | 설명 |
|---------|------|
| **정보 수집** | 필요한 정보를 얻기 어려움 |
| **정보 분석** | 수집한 정보를 해석하기 어려움 |
| **의사결정** | 매수/매도 결정을 내리기 어려움 |
| **실행** | 결정한 내용을 실행하기 어려움 |
| **모니터링** | 보유 종목/시장 상황 파악이 어려움 |
| **리스크 관리** | 손실 대비, 포지션 관리 어려움 |

## 활용 방법

1. 투자 중 불편함을 느낄 때마다 간단히 메모
2. 정기적으로 메모를 정리하여 파일로 작성
3. 기능 정의 시 우선순위 높은 문제부터 해결책 설계
