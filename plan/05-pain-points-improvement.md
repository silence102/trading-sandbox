# 투자 불편 사항 개선 계획

## 개요

실제 투자 진행하면서 경험한 불편 사항들과 개선 사항, 구현해야 할 기능에 대한 생각을 정리

> 기록 기간: 2026년 1월 — 한국투자증권 리포트 수기 분석 + 실전/모의 투자 병행

---

## 불편 사항 목록

### [1] 증권사 리포트 분석에 과도한 시간 소요

- **상황**: 한국투자증권의 유형별 리포트를 읽고 투자 인사이트를 도출하는 작업
- **문제점**:
  - 리포트 하나 분석에 최소 30분 이상 소요
  - 단일 증권사 리포트만으로는 다양한 투자 요인(거시, 수급, 뉴스 등)을 파악하기 부족
  - 시간 대비 얻을 수 있는 정보의 양과 다양성이 제한적
- **개선 방안**:
  - AI를 활용한 리포트 자동 요약 및 핵심 포인트 추출
  - 여러 정보 소스(DART 공시, 뉴스, 거시지표)를 통합한 종합 브리핑으로 단일 리포트 의존도 낮추기
  - 현재 진행 중인 일일 브리핑 자동화가 이 문제를 직접 해결하는 방향
- **우선순위**: 높음
- **연계**: `02-daily-briefing-expansion.md` — 증권사 리포트 요약 섹션 추가 검토

---

### [2] 수기 분석의 번거로운 디지털화 과정

- **상황**: 종이에 수기로 리포트 분석 메모 작성 → 스캔 → 업로드 → AI 분석
- **문제점**:
  - 작성 → 스캔 → 업로드의 3단계 마찰이 반복 작업 의욕을 저하시킴
  - 디지털화 전까지 분석 내용을 재활용하거나 검색하기 어려움
  - 스캔 품질에 따라 AI 분석 정확도가 달라짐
- **개선 방안**:
  - 수기 대신 구조화된 디지털 템플릿으로 분석 직접 입력
  - 종목별 분석 노트를 `notes/` 폴더 아래 마크다운으로 누적 관리
  - 향후 Structured Outputs 구현 시 AI가 종목 평가를 자동 저장 → 수기 필요성 감소
- **우선순위**: 중간
- **연계**: `01-skills-insight-application.md` — P3-A Structured Outputs

---

### [3] 투자 계획과 실제 실행 간의 괴리 (감정적 판단)

- **상황**: 분석을 통해 포트폴리오를 구성했음에도 실제 매수/매도 시 계획과 다른 행동을 함
- **문제점**:
  - 구성해둔 포트폴리오 비중 무시하고 즉흥적 매수
  - 계획에 없던 단기 트레이딩 감행 → 손실 발생
  - 이성적 판단이 흐려지는 순간(급등/급락 상황)에 규율이 무너짐
- **개선 방안**:
  - **투자 원칙 문서화**: 매수 전 체크리스트 마련 ("이 종목은 왜 사는가? 포트폴리오 내 비중은?")
  - **알림 시스템**: ±5% 이상 변동 시 계획 재확인 알림 (감정적 대응 방지)
  - **사전 목표가/손절가 기록**: 매수 시점에 목표가와 손절가를 notes에 기록하는 습관
  - **일일 브리핑 활용**: 아침 브리핑에서 보유 종목 현황과 원래 투자 근거를 함께 상기
- **우선순위**: 높음
- **연계**:
  - `02-daily-briefing-expansion.md` — 관심종목 목표가 도달 알림
  - `04-kis-api-customization.md` — 포트폴리오 분석, 수익률 계산

---

### [4] 투자에 과도한 주의·시간 집중

- **상황**: 본업 등 다른 할 일이 있음에도 투자 모니터링에 지나친 시간을 쓰게 됨
- **문제점**:
  - 장중에 시세를 자주 확인하게 되어 집중력 분산
  - 능동적 모니터링이 필요한 구조라 자동화 없이는 해결 어려움
  - 정보 확인 안 하면 불안, 확인하면 즉흥 매매 충동 — 악순환
- **개선 방안**:
  - **시간 분리**: 브리핑 확인 시간을 아침·마감 2회로 고정, 그 외 시간에는 확인 안 하는 규칙 수립
  - **자동 알림**: 관심 종목이 특정 조건(±5%, 목표가 도달 등) 충족 시에만 알림 → 불필요한 확인 감소
  - **미드데이 브리핑 활용**: 장중 점검을 브리핑 한 번으로 대체
- **우선순위**: 높음
- **연계**: `02-daily-briefing-expansion.md` — 알림 시스템, 급등/급락 알림

## 공통 개선 방향 요약

| Pain Point | 핵심 원인 | 자동화로 해결 가능 여부 |
|-----------|---------|-------------------|
| 리포트 분석 시간 과다 | 수동 정보 처리 | ✅ AI 요약 + 브리핑 통합 |
| 수기 디지털화 번거로움 | 아날로그-디지털 마찰 | ✅ 구조화 입력 + Structured Outputs |
| 계획-실행 괴리 | 감정적 판단 | 🔶 체크리스트 + 알림으로 보조 |
| 투자 시간 과집중 | 수동 모니터링 의존 | ✅ 조건부 알림 + 정기 브리핑 |
